# Implementation plans

One markdown file per unit of work, named with a short kebab-case slug (e.g.
`rides-apollo-events-api.md`). Claude drafts the plan here; **a human approves it
before any code is written** (see `../dev-workflow.md`, step 2).

A plan is the thing that gets reviewed -- it's diffable, referenceable, and outlives
any single Claude session. Keep approved plans in the repo so the PR can reference
them and reviewers can check the diff against the plan that was agreed.
